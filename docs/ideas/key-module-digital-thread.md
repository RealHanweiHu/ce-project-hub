# 关键模块数字主线：从 DRV 减负到轻量 PLM

> 状态：已确认方向
> 日期：2026-07-16
> 适用范围：DRV 创建、关键模块库、产品技术基线、BOM 维护与产品库信息架构

## Problem Statement

当前项目轴与产品轴虽然同时存在，但没有用同一组受控对象连接起来：

- DRV 创建时用自由文本填写“来源产品或模块、型号/版本、证据”，既费时，也无法可靠搜索、复用或追溯。
- 产品库把整机、普通零部件和可复用关键模块混在一起，模块缺少独立编号、内部 BOM、审批和使用范围。
- DRV 的“模块复用”只影响任务数量，没有与项目工作 BOM 形成同一事实来源，可能出现任务按模块 A 裁剪、BOM 却使用模块 B 的情况。
- 现有 BOM 支持逐行维护、冻结、差异和 where-used，但没有批量导入，也无法把关键模块从普通物料中可靠识别出来。
- 旧设计把所有技术变化都压进 Product Revision，导致项目、产品技术基线和包装/标签轻量改版的职责混淆。

用户真正需要的不是增加一套重型 PLM，而是：在不增加立项负担的前提下，让已经验证过的模块可以被准确选择、自动带入 BOM，并真正减少 DRV 的任务量。

## Recommended Direction

建立一条以“受控关键模块”为连接对象的轻量数字主线：

```text
关键模块（独立编号 + 固定内部 BOM + 审批）
        ↓ 可被搜索/选择
DRV 项目模块基线 ──同一事务── 项目工作 BOM
        ↓ 项目执行与验证
产品技术基线（项目完成后发布）
        ↓
产品页的配置、BOM、关联项目与使用历史
```

第一阶段只治理三个最有复用价值的物理模块：

1. 电池/能源模块
2. 机芯/电机/泵体等核心功能模块
3. PCBA 电子硬件模块

DRV 任务裁剪仍使用六个模块：电池、核心功能、电子硬件、软件与连接、结构与模具、ID/CMF。每个模块只有“复用 / 不复用”两种状态。前三个模块选择“复用”时必须引用一个已批准的关键模块；后三个模块第一阶段只记录复用状态，不建立模块主数据。

来源产品不是必填项。用户可以：

- 先选来源产品，自动带出该产品当前技术基线中的三个关键模块；
- 或直接跨产品品类搜索关键模块，同品类结果优先；
- 对自动带出的结果逐项覆盖。

## Key Assumptions to Validate

- “同一模块”的业务判断可以稳定落在内部受控 BOM 完全一致上，而不是依赖名称或说明文字。
- 工程师愿意创建并维护关键模块，产品经理或项目经理能够承担“允许项目使用”的批准责任。
- 三类物理模块覆盖了第一阶段最主要的复用价值，不需要立即把结构、软件、ID/CMF 全部主数据化。
- 现有项目工作 BOM 能作为模块选择的落点；模块基线与 BOM 可以在一个事务中提交。
- 当前数据均为测试数据，可以直接调整模型，无需设计历史数据迁移和兼容层。
- DRV 至少复用六个模块中的一个；全部不复用时应切换为 NPD，而不是允许绕过。

## MVP Scope

### 1. 关键模块库

- 三种模块类型、唯一模块编号、名称、品类、关键参数、状态和附件/证据引用。
- 模块内部 BOM；批准后组成不可原位修改。
- 任一受控子件变化时，通过“基于此模块新建”产生新模块编号，并保留来源和 BOM 差异。
- 工程师创建与技术确认；产品经理或项目经理批准项目使用。
- 模块搜索、详情、审批队列、where-used、限制使用和停用。

### 2. DRV 创建减负

- 保留项目基本信息和六模块二元选择，不显示项目目标、规格书引用、安全/法规确认等非必要输入。
- 来源产品可选；选择后自动带出可用关键模块。
- 三个物理模块支持按编号、名称、型号和关键参数搜索。
- 只有点击“创建项目”且六模块全部未复用时才弹窗，不在填写过程中持续提醒。
- 弹窗只提供“返回修改”和“切换为 NPD”；切换后保留基本项目字段，清除 DRV 复用数据。
- 服务端同样拒绝“零复用 DRV”。

### 3. 同一事实来源

- 创建项目时一次性写入不可变的项目模块基线快照，并同步项目工作 BOM。
- 后续来源产品发生 ECO，不回写或漂移已创建项目的基线。
- DRV 任务按六模块状态生成；模块引用与任务裁剪结果可追溯。

### 4. 产品轴重整

- 产品库入口重组为“产品 / 关键模块 / BOM 与配置”。
- 项目完成生成或更新产品技术基线；技术基线承载关键模块与冻结 BOM。
- Product Revision 只服务于包装、印刷、标签、文案等轻微改版，不再承载项目技术发布。
- 产品页展示当前技术基线、三个关键模块、BOM、关联项目和轻量 Revision。

### 5. BOM 批量导入

- 支持 Excel/CSV：选择工作表、字段映射、预览、校验、重复匹配、错误行下载和原子提交。
- 未匹配的关键模块行最多生成“候选草稿”，不得自动批准。
- 普通物料不会自动进入关键模块库。

## Not Doing

- 不建设完整企业级多层 PLM、ERP/MES 对接或跨系统物料主数据治理。
- 不为三个非物理模块建设主数据或内部 BOM。
- 不把供应商/二供变化纳入 DRV 模块判断；它走独立供应商审核、样件承认和切换流程。
- 不把仅包装、标签、文案变化作为项目；它进入产品库轻量 Revision。
- 不把重大包装运输或认证变化作为 DRV 模块；按影响程度走 ECO。
- 不在 MVP 中根据 BOM 差异自动恢复、保护或重排已经生成的任务；第一阶段只给影响建议。
- 不迁移历史项目或产品数据。
- 不允许批准后的模块直接改内部 BOM。

## Open Questions

这些问题不阻塞结构设计，可在实施前由管理员配置或在试用中确定：

- 三类模块编号的具体格式、号段与是否允许人工指定。
- 各模块类型的关键参数模板及品类标签词表。
- 技术确认与项目使用批准的时限、代理人和提醒策略。
- BOM 导入第一版优先兼容的客户表格模板。
- 产品技术基线的展示编号采用日期、项目号还是独立流水号。

# 响应式适配（手机/平板/桌面）设计规格

> 状态：已确认（用户批准设计）
> 分支：`feat-responsive`
> 日期：2026-06-24

## 1. 目标

让整个 app 适配 **手机 / 平板 / 桌面** 三档屏幕，同时**桌面端零回归** —— 现有桌面（Linear 改版后）布局像素级不变，只为小屏增量加自适应行为。

## 2. 核心原则（硬约束）

- **桌面冻结**：桌面（Tailwind `lg`/`xl`，≥1024px）的现有布局/外观**原样保留**。所有改动是**移动优先增量**：只新增 base / `sm` / `md` 行为，不改变 `lg:`+ 的呈现。
- **零回归验收**：每屏改完，用 preview 在 **1280px** 截图与改前比对，必须像素级一致；不一致即视为回归，必须修正。
- 无新业务逻辑、无数据层改动 —— 纯布局/CSS（Tailwind 响应式类）。

## 3. 断点

| 档 | 宽度 | Tailwind |
|---|---|---|
| 手机 | < 640px | base（无前缀） |
| 平板 | 640–1023px | `sm` / `md` |
| 桌面 | ≥ 1024px | `lg` / `xl` |

外壳已用 `lg` 作"图标栏收抽屉"的断点，全程沿用 `lg` 为"桌面"分界。

## 4. 现状（已核对）

- **外壳已响应式**：`Home.tsx` 的 60px 图标栏 `fixed lg:sticky` + `<lg` 抽屉化（汉堡 + 遮罩 `setSidebarOpen`）；顶栏面包屑/快捷键 `hidden sm:`；内容区 `p-4 lg:p-7`。`index.css` 无硬 `min-width` 锁死。
- **内容页参差**：ProductLibraryView / RequirementPoolPanel / ProjectDetailView 响应式类较多；**OverviewPage / PortfolioDashboard / MyTasksView / CalendarPage / ProjectListView 偏弱** —— 优先处理。

## 5. 逐屏适配模式

### 5.1 外壳（Home.tsx）— 小修补
- 已响应式，仅补：顶栏右侧元素在窄屏的间距/截断；详情/弹窗在手机改近全宽（`max-w-[calc(100vw-2rem)]` 之类）。桌面不动。

### 5.2 总览（overview/PortfolioDashboard + OverviewPage）
- 6 KPI 卡：手机 `grid-cols-2`、平板 `sm:grid-cols-3`、桌面 `lg:grid-cols-6`（桌面保持现 6 列）。
- 今日聚焦 3 项：手机 1 列、桌面 3 列。
- 两栏区（风险预警/组合进度 | 即将Gate/阶段分布）：手机/平板堆叠，`lg:` 并排（现状）。
- PortfolioMetricsTable：小屏外层 `overflow-x-auto`，表格 `min-w` 保表头不挤。

### 5.3 看板（ProjectListView）
- 看板视图：阶段列容器 `overflow-x-auto`，每列定宽（现状），手机横向滚动浏览 6 列。
- 列表视图：手机窄屏外层 `overflow-x-auto` 横向滚动整张表（保留全部列、不隐藏数据）；桌面表格不变。
- 顶部筛选/搜索/分组行：`flex-wrap`，手机换行。
- 详情弹窗（已是居中 Dialog）：手机 `max-w-[calc(100vw-1.5rem)]`，高度 `max-h-[85vh]` 已有。
- 泳道分组：泳道内每行阶段列同样横滚。

### 5.4 我的任务（MyTasksView / TaskListView）
- 任务行：手机内 优先级/截止日期/负责人 堆叠或缩；桌面行布局不变。
- 列表/看板段控 + 状态筛选 + 搜索：`flex-wrap`。

### 5.5 日历（CalendarPage）
- 5 张 stat 卡：手机 `grid-cols-2`、桌面现状。
- 月历网格：手机格子高度压缩、事件 chip 文本截断/转点；**窄屏（<640）默认切到「截止清单」更易读**（保留切回日历的能力）。
- 顶部 tab/「只看我负责的」/类型筛选下拉：`flex-wrap`。

### 5.6 项目详情（ProjectDetailView）
- 头部（名称/类型/风险/负责人/进度/推进）：手机换行堆叠。
- P1–P7 阶段 stepper：`overflow-x-auto` 横滚。
- 标签栏（概览/任务/…/变更记录 多标签）：`overflow-x-auto` 横滚。
- 各 tab 内的表格/网格：小屏横滚或堆叠。

### 5.7 产品库 / 需求池（ProductLibraryView / RequirementPoolPanel）
- 已较响应式，补缺口：产品卡网格列数（手机 1–2 / 平板 2–3 / 桌面现状）；大弹窗（RevisionsDialog）手机近全宽 + 内部滚动；内嵌表格横滚。

## 6. 架构 / 单元

- 纯前端布局改动，按"屏"为单元，每屏一个 task，互相独立。
- 不抽新组件（除非某段重复的响应式包裹值得提取）；优先在既有 JSX 上加 Tailwind 响应式前缀。
- 共享基元（LinearCard 等）保持桌面外观；如基元需小屏调整，用可选 prop，默认等于现桌面表现。

## 7. 数据流 / 错误处理

无变化（无逻辑/数据改动）。

## 8. 测试 / 验收

- `pnpm check`（tsc）保持通过。
- 每屏用 `preview_resize` 在 **390（手机）/ 768（平板）/ 1280（桌面）** 三档截图：
  - 手机/平板：不破版、不溢出横向滚动条（除非有意横滚区）、可点可读。
  - **桌面 1280：与改前像素级一致（零回归）—— 硬验收标准。**
- 全仓 0 残留 stone/amber/ce-*（不在响应式改动里引入旧类）。

## 9. 实施顺序

先做响应式弱的：总览 → 看板 → 我的任务 → 日历 → 项目详情 → 产品库/需求池 → 外壳小修补 → 全局三档走查。逐屏提交。

## 10. 非目标（YAGNI）

- 不重做任何页的信息架构/交互（只调布局自适应）。
- 不为手机做专属精简视图（除日历窄屏默认切清单这一处）。
- 不动业务逻辑、数据、tRPC。
- 不引入响应式相关的第三方库。

## 11. 不回归

- 桌面（≥1024）所有页外观零变化（硬标准）。
- 已完成的 Linear 视觉、看板 Phase 2、各功能修复 不受影响。

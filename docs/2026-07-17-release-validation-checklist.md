### 1.2 缺陷等级

| 等级 | 定义 | CE Project Hub 示例 | 发布规则 |
|---|---|---|---|
| P0 | 数据丢失、越权、安全问题、核心系统不可用、不可恢复的错误 | 无法登录；跨项目读取文件；Gate/Release 可绕过；迁移删错数据；阶段推进产生半成功状态 | 必须修复并回归，未关闭即 `NO-GO` |
| P1 | 核心流程不可完成且无可靠替代路径 | 无法创建项目、完成任务、评审交付物、处理 Issue、发布产品；移动端关键操作不可达 | 原则上必须修复；例外需负责人书面接受 |
| P2 | 局部功能或体验问题，有明确替代路径 | 筛选状态不清晰、错误提示不足、局部布局溢出 | 可带风险上线，但必须入计划 |
| P3 | 文案、视觉或低频易用性问题 | 对齐、措辞、轻微样式不一致 | 可进入优化池 |

### 1.3 Go / No-Go 门槛

- [ ] P0 未关闭数为 0。
- [ ] P1 未关闭数为 0；若有例外，已记录影响、替代方案、责任人、修复日期和接受人。
- [ ] 部署前在隔离预生产完成 SMK-01～15，均为 `通过` 或已按规则处理的 `通过（有观察）`，无 `失败/阻塞/未测`。
- [ ] 切流后完成 FAST-01～07，全部 `通过`；任何 `失败/阻塞/未测` 立即停止放量并评估回滚。
- [ ] 核心角色的正向流程和越权负向流程 100% 通过。
- [ ] 类型检查、自动化测试、生产构建、全新库迁移和存量库升级预演全部通过。
- [ ] 无未解释的数据差异、半成功状态、重复记录、孤儿文件或越权访问。
- [ ] 生产备份可恢复，代码与数据回滚路径均已演练。
- [ ] 真实浏览器、真实对象存储，以及启用时的真实钉钉链路已有证据。
- [ ] 已知问题、监控、值班人和回滚触发条件已确认。

### 1.4 两阶段发布决策

1. **部署前决策**：在隔离预生产完成 SMK-01～15、适用 P0/P1、迁移预演、备份恢复和回滚演练，单独记录 `GO / CONDITIONAL GO / NO-GO`。
2. **部署后决策**：切流后完成 FAST-01～07，并在 30 分钟和 24 小时分别给出 `继续放量 / 维持观察 / 回滚 / 最终关闭发布`。预先填写 5xx、登录失败、DB/S3 错误、队列积压、内存和慢查询阈值，以及 RTO、RPO 和最迟回滚时限。

## 2. 当前上线前必须优先核实的风险

以下是基于当前仓库的静态审计结果，不是泛化建议。未验证或未处理前，应视为发布风险。

| ID | 等级 | 风险 | 发布前要求 |
|---|---|---|---|
| RSK-01 | P0 | `scripts/test.mjs` 会优先使用 `TEST_DATABASE_URL`，否则使用 `DATABASE_URL`，并对任何可达数据库执行迁移和有写入/删除的集成测试 | 未增加测试库护栏前，禁止在加载生产 `.env` 的环境运行 `pnpm test`；只允许显式独立测试库 |
| RSK-02 | P0 | `scripts/deploy-ecs.sh` 当前先启动新应用再迁移，后台任务可能在新表尚未创建时运行 | 发布前必须修改实际脚本/操作手册为先迁移后切流量，或在迁移期间硬性阻断新应用流量与后台任务；只做副本预演不足以关闭风险 |
| RSK-03 | P0 | 部署脚本的 `curl` 未强制检查 HTTP 失败，500 也可能让烟测假绿 | 烟测必须断言 HTTP 状态、响应内容、DB、S3 和后台心跳 |
| RSK-04 | P0 | `0044_true_medusa.sql` 会重建枚举并删除孤儿记录，仓库没有 down migration | 用脱敏生产快照预演；导出删除清单并由业务确认；准备 DB 与对象存储快照恢复 |

| RSK-06 | P0 | `.env.example` 表示注册默认关闭，但服务端变量缺失时可能默认开放 | 验证变量缺失、false、true+邀请码三种情况；生产必须显式配置 |

| FX-ACCESS-A/B | 两个项目、同一用户拥有不同角色、文件受众齐全 | 17 角色矩阵、跨项目 IDOR、文件隔离 | 重新 seed；保留基线行数和对象 key 清单 |
| FX-GATE-BLOCK-* | 每个项目只缺一种 readiness 条件 | SMK-06 / GATE-06 阻塞矩阵 | 每个缺口单独项目；执行后销毁并重建 |
| FX-GATE-READY-* | Gate 全满足、尚未决策 | 并发、双击、原子性、approved/conditional/rejected | 每种结论与故障独立克隆，禁止重复利用 |
| FX-QA-* | EVT/DVT/PVT 或类别对应验证阶段；含测试、Issue、NPI、签样数据 | QA-01～10 | 固定 seed 脚本或数据库快照恢复 |
| FX-RELEASE-* | 接近 Release，分别缺产品/P0P1/证据/override，另有 ready 项目 | SMK-11 / GATE-12 | 每个分支独立项目；成功发布后重新 seed |
| FX-FILE-* | 空 bucket 前缀、文件记录基线为 0 | 上传边界、受众、S3/DB 半成功 | 每轮使用唯一前缀；结束后 DB/S3 双向对账并清理 |
| FX-PERF | 固定随机种子生成 100、500、峰值项目 | PERF-01～05 | 保存生成参数；每轮恢复同一快照并预热 |

故障注入规则：

- 401/403 使用真实过期会话或低权限账号，并同时验证 UI 与直接 API；不要用前端假响应替代权限测试。
- 404 使用不存在 ID 和跨项目实体 ID；结果不得泄露对象是否存在。
- 500、延迟和“上游已提交但响应被丢弃”使用仅限隔离环境的故障代理或测试开关，并记录注入点。
- 断网使用浏览器 Offline 或代理断连；超时应超过客户端 30 秒上限。
- DB/S3 故障只在隔离环境通过停服务、撤销临时凭据或故障代理注入，禁止操作生产依赖。
- 每条破坏性用例记录创建/修改/删除的实体 ID、request/correlation ID、执行前后关键表行数和对象 key。
- 复位后先跑基线校验；基线不一致时后续用例标记为“阻塞”，不能继续并给出假通过。

- 安全：17 角色矩阵、跨项目 IDOR、文件受众隔离、脚本注入和 secret 泄漏。

### 9.3 当前明显缺口

- 客户端 TSX 不在 Vitest 包含范围，没有组件级自动化回归。
- 没有 Playwright/Cypress 浏览器 E2E 门禁。
- 没有 HTTP/反向代理、真实 multipart、真实对象存储自动化。
- 没有 coverage 阈值、负载测试和仓库内 CI workflow。
- 测试入口缺少“必须为本地/测试库”的硬护栏，在修复前不能安全地直接执行全量测试。
- 健康检查未充分验证数据库、迁移版本和对象存储。
- 迁移元数据与当前 journal/schema 存在漂移信号。

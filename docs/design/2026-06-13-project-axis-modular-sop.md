# 项目轴细化 — 模式 × 对象 × 模块化 SOP（v2 · 平台化）

> 状态：**框架锁定 v2** · 日期：2026-06-13 · 配套：`2026-06-13-two-axis-plm-architecture.md`
> v2 参考 Anker / 华为 等平台化 NPD 实践修订：用 5 级变更等级替代二元复用/重做、Gate 强制守门人、NPD 三细分、平台/核心模块集、认证矩阵、角色统一。
> 用工厂现有词汇 **NPD / ECO / IDR** 表达。

---

## 1. 一个项目 = 三维 + 一个等级

```
项目 = 模式(NPD/ECO/IDR) × 对象(整机/零部件) × { 每个模块的变更等级 }
```

- **模式**：NPD（三细分，见 §2）/ ECO / IDR。
- **对象**：整机产品 / 零部件（机芯、电机…）。「机芯开发」= 对象为零部件，不是独立模式。
- **变更等级**（替代二元复用/重做）：每个模块声明 **完全复用 / 复用+兼容验证 / 小改 / 重做 / 新开发**，决定该模块的任务深度（见 §4）。

---

## 2. 开发模式

**NPD 三细分（平台化打法，避免"全新=全做"）：**

| 子模式 | 含义 | 默认模块等级 |
|---|---|---|
| **新品类 NPD** | 做一个没做过的品类（通常含新平台） | 全部 新开发/重做 |
| **新平台 NPD** | 新平台，沉淀可复用的平台模块集 | 平台模块 新开发；产品专属 重做 |
| **平台派生 NPD** | 现有平台派生新整机（大厂最高频） | 平台模块 完全复用/兼容验证；外壳·ID·认证·包装 重做 |

| 模式 | 驱动 | 典型等级分布 |
|---|---|---|
| **ECO** | 工程变更：降本/质改/功能/法规 | 只动被变更影响的模块，等级按影响程度定 |
| **IDR** | 外观 / ID | 平台/核心 复用+兼容验证；外壳 重做；认证 按目标市场重取；ID 越复杂越向 NPD 滑 |

**对象 × 模式：**

| | 整机产品 | 零部件（机芯/电机） |
|---|---|---|
| NPD | 全新/派生整机 | 全新机芯开发 |
| ECO | 整机工程变更 | 机芯改进 |
| IDR | 外观翻新 | —（零部件无外观） |

**ECO 术语区分：** ECO 作为**项目模式**（项目轴）= 工程变更驱动的项目；ECR→ECO→ECN 作为**变更工作流**（产品轴）= 请求→批准→通知。一条变更经 ECR 批准为 ECO，ECO 触发一个 ECO 模式的项目，完成发 ECN。**ECO 是两轴连接点。**

## 3. 品类 + 平台层

- **品类**：开放可增长，seed 风扇 / 充气泵 / 车载清洁…；新品类 NPD 可新建。
- **平台**：可复用模块集合（机芯 + 电机 + 电池包 + 充电 + 固件 + 架构）的**版本化捆绑**；整机派生自平台。这是平台派生 NPD 与 IDR 高复用的根基。
- **三层闭环**：版本层（项目→发布→新 Rev，产品迭代）/ 平台层（一平台派生多产品）/ 品类层（新品类扩张产品线）。

## 4. 变更等级（替代二元复用/重做）— 核心修订

| 变更等级 | 含义 | 任务深度 |
|---|---|---|
| **完全复用** carry-over | 接口/边界不变，原样沿用 | 仅引用版本，0 任务 |
| **复用+兼容验证** reuse+verify | 不改设计，但环境/接口/相邻模块变了 | **兼容性验证集**：热路径、跌落、装配/固定、防护(IP/充电口)、EMC… 无设计任务 |
| **小改** minor | 局部修改 | 局部设计 + 针对性验证 |
| **重做** redesign | 整体重新设计 | 完整任务块 |
| **新开发** new | 此前不存在 | 完整任务块 + 定义 |

**关键原则：复用 ≠ 免验。** 例：充气泵换外壳，电池包设计没变 → 等级 = **复用+兼容验证**，热路径 / 电池仓固定 / 充电口防护 / 跌落必须重验，不能一条确认带过。

**模块依赖带动：** 一个模块的变更会**带动相邻模块**至少升到「复用+兼容验证」。换外壳 → 电池/热/防护被带动。v1：人工声明每模块等级 + 系统给「受影响项」提示清单；v2 后续：模块依赖图自动联动。

## 5. Gate 模型 — 强制守门人 + 模块责任

**会签盘 = 强制守门人（恒签） ∪ 活跃模块责任职能（条件签）。**

| 守门人 | 始终会签的内容 |
|---|---|
| 产品经理 | 产品定义/范围/商业目标 |
| 项目经理/PMO | 计划/节点/资源/风险推进 |
| QA（+cert） | 兼容性/可靠性/认证就绪 |
| 工艺·设备 pe + 生产 mfg | 量产风险/工艺就绪 |
| SCM | 供应/物料/成本就绪 |
| **电池安全**（锂电产品） | 电池安全/运输/滥用测试 |

> **即使电池/PCBA/认证模块是「完全复用」，守门人仍必须签兼容性/供应/量产风险。** 安全与合规绝不因「复用」被跳过——这是安全/合规类产品的硬底线。

**一个阶段可挂多道命名评审，每道 = 会签盘 + 检查清单：**
- **ID 评审** — 研发 + sales + manager；检查项：符合 PI、人机、外观成本…
- **DFM 评审** — pe；检查项：可制造性、模具可行性…
- **安全评审**（锂电/高压/高速电机）— battery_safety + qa；检查项：电池滥用、过流过温保护、防护…
- **认证评审** — qa/cert；检查项：按 §6 认证矩阵展开。

## 6. 模块库

### 共享模块（跨品类，骨架稳定可增长）

| 模块 | 责任职能 | 关键检查点 |
|---|---|---|
| 外壳 / 模具 | rd_mech + pe | ID 评审、DFM、开模、模具验收（见样板） |
| PCBA / 固件 | rd_hw + rd_sw | 原理图、Layout、打样、EMC/可靠性、SMT 治具 |

| 工艺 / 治具 | pe + mfg | 工艺文件、治具/模具、产线导入、节拍/良率 |

### 平台 / 核心模块集（每品类一组，非单个）

| 品类 | 平台/核心模块集 |
|---|---|
| 锂电充气泵 | 气泵机芯 · 电机 · 压力传感/算法 · 电池包·BMS·充电 · 热设计 · 外壳风道 · PCBA/固件 |
| 车载吸尘器 | 高速电机·风机 · 风道 · 尘杯·滤芯 · 电池包 · 充电 · 热·噪音·密封 · 附件 |
| 风扇 | 电机 · 扇叶·风道 · 电池·充电 · PCBA/固件 |
| 手动泵 | 机械泵体 · 结构 |

> 模块库 = 共享模块 + 各品类平台/核心模块集。各模块完整任务块（任务×阶段×责任职能×门禁）待逐模块对齐后填全。

### 认证矩阵（targetMarkets × 变更 → 检查项）

项目/产品设 `targetMarkets`（EU/US/UK/JP/CN…）。认证模块的检查项由市场与变更自动展开：

| 市场 | 法规集（示例） |
|---|---|
| EU | CE（EMC/LVD，无线→RED）、RoHS、REACH、电池指令、包装指令、多语说明书 |
| US | FCC、UL/ETL、CA Prop 65、电池 UN38.3 运输 |
| UK | UKCA |
| JP | PSE、技适(MIC，无线) |
| CN | CCC/能效（如适用） |

**变更触发：** 无线变→RED/FCC 重测；电源/充电变→安规重测；电池变→UN38.3+电池指令；材料变→RoHS/REACH/CA65；标签/说明书/包装变→对应市场标签法规。

### 已对齐模块：气泵机芯（零部件产品 · 平台核心）

> 机芯身兼两职：① 作为**零部件产品**被开发（自己出 Rev）；② 作为整机里的**复用模块**被 BOM 引用（where-used）。机芯/电机边界按「机芯含电机匹配、电机本身为相邻模块」假设，可调。

**① 作为零部件产品（开发它自己）：**

| 阶段 | 任务 | 执行方 | 内部责任 | 门禁 |
|---|---|---|---|---|
| 需求 | 性能目标：气压/流量/噪音/寿命/温升/气密/电压功率/尺寸接口 | 内部 | rd_mech + qa | |
| 设计 | 泵体设计（缸体/活塞或膜片/进排气阀/传动） | 内部 | rd_mech | |
| 设计 | 电机匹配/选型（扭矩·转速·效率） | 内部/供应商 | rd_hw + rd_mech | |
| 设计 | DFM（泵体注塑/装配可行性） | 内部 | pe | ✓ 设计 Gate |
| 样件 | 机芯样件试制、性能初测 | 供应商+内部 | rd_mech + qa | |
| 验证 | 性能/寿命/噪音/气密验证 | 内部/实验室 | qa | ✓ 验证 Gate |
| 验证 | 关键供应商/产能确认（电机·气阀·密封件） | 内部 | scm | |
| 定型 | 小批试制、工艺/治具、定型 release | 内部 | pe + mfg + qa | ✓ 定型 Gate（出机芯 Rev） |

**② 作为整机复用模块（被引用，按变更等级）：**

| 变更等级 | 场景 | 任务 |
|---|---|---|
| 完全复用 | 同平台、接口/风道不变 | 引用机芯 Rev X，0 任务 |
| 复用+兼容验证 | 新整机/新外壳/新风道 | 集成兼容验证（装配·气路密封·整机噪音·热·振动），不重测机芯固有性能 |
| 小改 | 接口/功率档适配 | 局部适配 + 针对性验证 |

**验证分层（模型补充）：** 机芯**固有性能**（气压/流量/噪音/寿命/气密）在机芯项目验证一次、随机芯 Rev 冻结成报告；整机复用时**只做集成验证**，引用机芯报告不重测 → 强化了「发布快照含测试报告」。

**where-used：** 机芯出新 Rev → 反查引用它的整机/平台 → 各自显式决策是否采纳（采纳则集成兼容验证），**不自动替换**。

**平台 vs 组件：** 机芯 = 组件产品（有 where-used）；平台 = 一组组件版本的捆绑（机芯+电机+电池+充电+固件），平台派生 NPD 继承之。

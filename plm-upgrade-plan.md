# CE Project Hub：向工厂级 PLM 系统的升级蓝图

本项目将从一个“轻量级项目管理工具”全面升级为支撑工厂业务流的 **PLM (Product Lifecycle Management，产品生命周期管理)** 系统。本次规划将所有业务痛点与技术债务整合，覆盖产品从立项、研发、试产（EVT/DVT/PVT）、量产（MP）到后续迭代的全生命周期。

---

## 第一部分：业务场景深度改造（工厂 PLM 核心）

工厂项目的核心不在于“任务打勾”，而在于物料、质量、文档与责任的强闭环。

### 1. BOM 与 ECN 物料协同（核心模块）
在工厂场景中，BOM（物料清单）和 ECN（工程变更通知）是业务流转的血液。
* **BOM 管理**：建立独立的 BOM 模块，支持 BOM 版本控制、关键物料标记、长周期物料识别。
* **供应链协同**：引入供应商字段与替代料（Alternate Parts）管理，跟踪成本变化。
* **ECR/ECN 闭环**：将 ECR（变更请求）与 ECN（变更通知）与任务和物料深度绑定，实现“变更发起 -> 影响评估 -> 审批 -> BOM 更新 -> 试产验证”的全流程闭环。

### 2. 问题管理升级为 8D/CAPA 闭环
工厂的质量问题（Issue）不能仅作为记录，必须遵循严格的质量体系。
* **流程重塑**：将 Issue 列表升级为符合 8D（8 Disciplines）或 CAPA（纠正与预防措施）标准的闭环流程。
* **字段强化**：增加问题发现、责任部门、临时措施（Containment）、根因分析（Root Cause）、永久对策、验证结果、关闭审批、复发记录等字段。

### 3. 文件交付物独立治理（文档管控）
工厂项目极度依赖文档版本，文件不能仅作为“附件”存在。
* **阶段必交物**：结合项目阶段（如 PRD、BOM、图纸、测试报告、认证文件、试产报告），设定各 Gate 评审的“必交物清单”。
* **独立文件系统**：设计独立的文件表结构，包含：版本号、所属对象、审批状态、存储 key、文件 hash、上传人、是否最新版（Is Latest）。实现历史版本可追溯，确保产线使用的始终是最新受控图纸。

### 4. 任务责任追踪强化
任务执行必须具备强约束力。
* **执行字段增强**：增加协作人、前置依赖（Finish-to-Start）、延期原因说明、完成证据上传、风险等级、审批状态。
* **逾期自动升级**：任务一旦逾期，系统应自动触发升级（Escalation）机制，抄送至上级主管。

## 第二部分：管理与协同体验优化

### 1. 管理层看板（项目健康度中心）
为管理层提供全局视角，避免陷入细节。
* **健康度指标**：展示项目健康度评分、延期项目预警、P0/P1 严重问题概览、各阶段 Gate 卡点。
* **资源与趋势**：提供研发资源负载分析、阶段通过率统计、风险热力图以及下周重点事项（Lookahead）。

### 2. 看板视图与拖拽交互
为 PM 提供高效的日常调度工具。
* **Kanban 视图**：支持按任务状态（Status）或负责人（Assignee）列展示。
* **交互升级**：支持拖拽（Drag & Drop）卡片改变状态或重新分配任务。

### 3. 通知与升级机制（异步队列）
工厂场景下，信息的及时触达至关重要。
* **多渠道推送**：集成企业微信/飞书/钉钉 Webhook，支持邮件提醒。
* **关键事件**：包含任务到期提醒、逾期自动升级、问题严重度升级、Gate 评审预警、项目日报/周报自动推送。

## 第三部分：底层技术与架构治理（基石）

业务复杂度的提升必须有稳固的技术底座支撑。

### 1. 数据库约束与审计事件化
* **数据库治理**：在 Drizzle + MySQL 基础上，补齐外键关系、索引优化、唯一约束。实施软删除策略（Soft Delete）与级联归档机制。针对大表（如日志、任务）提供游标分页（Cursor Pagination）查询能力。
* **全量审计日志**：所有关键操作（创建、修改、删除、审批、上传文件、变更 BOM）必须写入审计日志。记录：旧值、新值、操作者、时间、来源（Source）。

### 2. 异步任务与通知队列
* **架构解耦**：引入任务队列（如 Redis 配合 BullMQ，或数据库队列表），将通知推送、报表生成、大文件处理、AI 总结等耗时操作从 HTTP 请求中剥离，改为后台 Job 异步执行。

### 3. 前端架构拆分与缓存治理
* **页面解耦**：当前 `Home.tsx` 承担过多业务编排。需逐步拆分为页面级容器、业务 Hooks、纯展示组件。
* **按需加载**：对低频重量级模块（Admin、Gantt、SOP）使用 `React.lazy()` 进行代码分割。
* **缓存优化**：统一 tRPC 缓存策略，建立层级 query key，利用 prefetch + invalidate 精确失效，降低数据库负载。

### 4. 测试分层与部署规范
* **测试体系**：在现有后端测试基础上，补充：权限测试、项目完整流程（Golden Path）测试、文件版本测试、8D 问题关闭流程测试，以及前端关键页面的冒烟测试（Playwright）。
* **生产部署**：明确生产环境的 `.env` 规范、数据库迁移（Migration）策略、备份恢复预案、日志与错误追踪（Sentry）接入、权限初始化脚本，以及管理员账号的安全创建流程。

## 实施阶段划分 (Phased Rollout)

| 阶段 | 重点目标 | 包含模块 |
|------|----------|----------|
| **Phase 1: 基础设施与底座** | 稳固底盘，消除技术债 | 数据库约束、审计日志、前端代码拆分、异步队列搭建 |
| **Phase 2: 核心业务重塑** | 匹配工厂 PLM 核心流 | BOM/ECN 模块、8D/CAPA 问题闭环、文件交付物独立治理 |
| **Phase 3: 执行与协同强化** | 提升执行力与沟通效率 | 任务责任追踪强化、看板视图、通知与升级机制 |
| **Phase 4: 管理与洞察** | 为决策层提供数据支撑 | 管理层看板、项目健康度中心、测试分层与部署规范 |
